AWS Basic Networking

Understanding Basic Networking Resources in AWS - The base for compute solutions on AWS

When designing cloud infrastructure, networking is the backbone. In AWS, setting up a secure and scalable network starts with a Virtual Private Cloud (VPC). In this post, we’ll walk through the essential networking components in AWS and how to provision them using Terraform.

Amazon Web Services (AWS) provides a robust set of networking components that allow cloud architects to design secure, scalable, and highly available infrastructure. In this post, we’ll explore the foundational networking resources in AWS and how they work together to enable connectivity and isolation.

Virtual Private Cloud (VPC)

A VPC is a logically isolated section of the AWS cloud where you can launch AWS resources in a virtual network that you define. You control IP address ranges, subnets, route tables, and gateways.

Public and Private Subnets

Subnets divide your VPC’s IP address range into smaller segments:

  • Public Subnets: Subnets that have a route to the internet via an Internet Gateway (IGW).
  • Private Subnets: Subnets without direct internet access, typically used for backend services or databases.

Route Tables and Routes

A route table contains a set of rules, called routes, that determine where network traffic is directed. Each subnet must be associated with a route table.

Example route for a public subnet: 


Destination: 0.0.0.0/0
Target: igw-xxxxxxxx

Internet Gateway (IGW)

An Internet Gateway is a horizontally scaled, redundant, and highly available VPC component that allows communication between instances in your VPC and the internet.

To make a subnet public, you must:

  • Attach an IGW to your VPC
  • Update the route table to direct traffic to the IGW
  • Ensure instances have public IPs

NAT Gateway

A NAT Gateway enables instances in a private subnet to connect to the internet or other AWS services, but prevents the internet from initiating connections with those instances.

NAT Gateways are:

  • Managed by AWS
  • Highly available within an Availability Zone
  • Used in conjunction with Elastic IPs

Conclusion

Understanding these core networking components—VPCs, subnets, route tables, IGWs, and NAT Gateways—is essential for designing secure and efficient cloud architectures in AWS. With these building blocks, you can tailor your network to meet the needs of any application, from public-facing web apps to private backend services.
