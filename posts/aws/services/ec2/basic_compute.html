Understanding Basic Public Compute Resources in AWS

Amazon Web Services (AWS) offers a wide range of compute resources that enable users to deploy scalable and secure applications. In this post, weâ€™ll explore the foundational public compute components in AWS and how they interact within a networked environment.

Amazon EC2 (Elastic Compute Cloud)

EC2 is the core compute service in AWS, allowing users to launch virtual machines with customizable configurations. EC2 instances can run various operating systems and are available in multiple instance types to suit different workloads.

Elastic Network Interfaces (ENIs)

An ENI is a virtual network card attached to an EC2 instance. It includes attributes like private IP addresses, security groups, and MAC addresses. ENIs can be detached and reattached to other instances, providing flexibility in network configurations.

Elastic IP Addresses (EIPs)

EIPs are static public IP addresses that can be associated with EC2 instances or ENIs. They allow instances to be reachable from the internet and can be remapped to other resources in case of failure or reconfiguration.

Elastic Block Store (EBS)

EBS volumes provide persistent block storage for EC2 instances. They can be attached to one instance at a time and are ideal for storing data that must persist beyond the life of the instance. EBS volumes come in various types optimized for performance and cost.

Security Group Rules

Security groups act as virtual firewalls for EC2 instances. They control inbound and outbound traffic based on rules defined by protocol, port range, and source/destination IP. Security groups are stateful, meaning return traffic is automatically allowed.

SSH Key Requirements

To access a Linux-based EC2 instance remotely, AWS requires an SSH key pair. The public key is stored in AWS and injected into the instance during launch, while the private key remains with the user. This key is used to authenticate SSH sessions securely.

  • You must create or import a key pair in the AWS EC2 console or CLI.
  • Ensure the private key file (usually .pem) is stored securely and has proper file permissions.
  • Use the key with an SSH client to connect to the instance: ssh -i my-key.pem ec2-user@

Networking Requirements

To deploy a public EC2 instance, the following networking components are typically required:

  • A VPC with a public subnet
  • An Internet Gateway (IGW) attached to the VPC
  • A route table with a route to 0.0.0.0/0 via the IGW
  • A security group allowing inbound traffic (e.g., SSH on port 22)
  • An Elastic IP associated with the instance or its ENI

Conclusion

AWS provides a robust set of public compute resources that can be tailored to meet the needs of any application. Understanding how EC2 instances interact with ENIs, EIPs, EBS volumes, security groups, and SSH keys is essential for building secure and scalable cloud infrastructure.
