Getting Started with the AWS Terraform Provider

Terraform is a powerful Infrastructure-as-Code (IaC) tool that enables cloud architects to define and provision infrastructure using declarative configuration files. One of its most widely used providers is the AWS provider, which allows users to manage AWS resources seamlessly. In this post, weâ€™ll explore how to configure the AWS provider, understand its authentication methods, and walk through a basic example.

Authentication Methods

The AWS provider supports several authentication mechanisms to securely access your AWS account:

  • Environment Variables: Set AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY in your shell environment.
  • Shared Credentials File: Use the ~/.aws/credentials file with named profiles.
  • EC2 Instance Metadata: Automatically retrieves credentials from the instance metadata service when running on EC2.
  • IAM Roles for Service Accounts (IRSA): Used in EKS clusters to assign IAM roles to pods.
  • Static Credentials in Provider Block: Not recommended for production due to security risks.

Example: AWS Provider Block


terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 6.0"
    }
  }
}

# Configure the AWS Provider
provider "aws" {
  region     = "us-west-2"
  access_key = "my-access-key"
  secret_key = "my-secret-key"
}

# Create a VPC
resource "aws_vpc" "example" {
  cidr_block = "10.0.0.0/16"
}

This configuration tells Terraform to use the AWS provider in the us-west-2 region and the default profile from your credentials file.

Example: Creating an S3 Bucket


resource "aws_s3_bucket" "example" {
    bucket = "my-terraform-example-bucket"
    acl    = "private"

    tags = {
        Name        = "MyBucket"
        Environment = "Dev"
    }
}

This resource block creates a private S3 bucket named my-terraform-example-bucket with tags for identification.

Conclusion

The AWS Terraform provider is a robust and flexible way to manage AWS infrastructure. By understanding its authentication options and mastering its configuration syntax, you can automate deployments and maintain consistent environments across your cloud architecture.

For more information, check out the AWS Terraform Provider documentation in the Terraform Registry
